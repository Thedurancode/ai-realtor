## API Commands

### Add Note
```bash
curl -X POST http://localhost:8000/property-notes/ \
  -H "Content-Type: application/json" \
  -H "X-API-Key: nanobot-perm-key-2024" \
  -d '{
    "property_id": 1,
    "content": "Your note here",
    "source": "manual"
  }'
```

### Get Notes
```bash
curl http://localhost:8000/property-notes/?property_id=1 \
  -H "X-API-Key: nanobot-perm-key-2024"
```

### Update Note
```bash
curl -X PUT http://localhost:8000/property-notes/5 \
  -H "Content-Type: application/json" \
  -H "X-API-Key: nanobot-perm-key-2024" \
  -d '{"content": "Updated note"}'
```

### Delete Note
```bash
curl -X DELETE http://localhost:8000/property-notes/5 \
  -H "X-API-Key: nanobot-perm-key-2024"
```

## Best Practices

### ‚úÖ DO
- Be specific: "Seller wants $850k firm" (not just "price info")
- Include dates: "Roof replaced 2023" (not just "new roof")
- Add context: "Buyer pre-approved $900k, looking in Brooklyn"
- Update regularly
- Use categories

### ‚ùå DON'T
- Vague notes like "follow up" (when? why?)
- Duplicate information
- Outdated notes (update or delete)
- Irrelevant opinions
- Sensitive financial info
